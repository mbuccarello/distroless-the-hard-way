# Python Distroless Architecture

This document describes the build architecture for the Python 3.14 distroless OCI image, orchestrated by the **Distroless Engine**. This architecture eliminates ABI (Application Binary Interface) inconsistencies and persistent segmentation faults by building the entire dependency graph from source.

## 🏗️ High-Level Overview

The architecture follows a "Clean Room" build approach where every library, from the core `zlib` to the final `python` binary, is compiled in an isolated environment with strictly controlled flags.

## 🛠️ Components

The Python stack utilizes the following core components, all built from source to ensure absolute transparency and ABI alignment:

1.  **Foundation Layer (CC)**:
    *   **Glibc**: The core C library.
    *   **OpenSSL**: For secure networking and cryptography.
    *   **Zlib & Bzip2**: For compression support.
    *   **LibFFI**: Required for Python's `ctypes` module.
    *   **SQLite & Expat**: For database and XML processing.
    *   **Ncurses & Readline**: For interactive shell support (available in `:debug` variants).

2.  **Runtime Layer**:
    *   **Python 3.14**: Compiled with `--enable-optimizations` and `--with-lto` against the foundation libraries.

## 📁 Filesystem & ABI Standards

The architecture enforces a strict **Unified FHS** policy:
*   **Library Path**: All shared objects (`.so`) are located in `/usr/lib`.
*   **Symlink Compatibility**: `/lib` and `/lib64` are symlinked to `/usr/lib` to ensure the ELF interpreter can resolve dependencies across all Linux kernel variants.
*   **Networking**: NSS libraries (`libnss_dns.so.2`) are included in the `base` image to ensure functional DNS resolution.

## 🛡️ Security & Compliance

*   **Zero-Executable Standard**: The production Python image contains no shell (`/bin/sh`) or troubleshooting utilities.
*   **Debug Variants**: Troubleshooting tools (Busybox) are only available in the `:debug` tagged images.
*   **License Extraction**: Licenses for all components are automatically harvested and stored in `/usr/share/doc/`.
*   **Keyless Signing**: The final image is signed via Sigstore/Cosign for non-falsifiable verification.
